# 推敲チェックリスト

生成・推敲した日本語技術文書を、提示前にこのリストで点検する。全文を一度通読し、違反を洗い出してから直す。詳細な根拠は [rules.md](rules.md) を参照。

## 整形

- [ ] 一文ごとに改行し、段落は空行で区切っているか
- [ ] 日本語中の半角英数字の前後に半角スペースを入れているか（約物に隣接する場合は除く）
- [ ] コード・ログ・コマンドはコードブロックに入っているか
- [ ] 用語の初出は太字、以後は「」になっているか
- [ ] ダッシュ・中黒を濫用していないか
- [ ] 見出しが単一トピックに絞られているか

## 論証

- [ ] 一段落一トピックを守っているか
- [ ] 段落冒頭で内容を示し、接続表現で前段落とつないでいるか
- [ ] 論証が一方向で、結論を一度だけ述べているか
- [ ] 誤読しやすい点を、明示的に否定してから説明しているか

## 厳密さ（最優先）

- [ ] 根拠なき推量を削り、かつ必要な不確実性は残しているか
- [ ] 異なるものを同一視していないか
- [ ] 複数要因を単一原因に還元していないか
- [ ] 概念の定義が章をまたいで一致しているか
- [ ] 因果を機構まで説明しているか
- [ ] 「必ず」「常に」など絶対化していないか

## 読み手の負荷

- [ ] 論証に不要な固有名詞を出していないか
- [ ] 抽象表現に括弧で補足があるか
- [ ] 新しい例の必要性を前置きしているか
- [ ] 冒頭で詳細を盛りすぎていないか

## 視点と語り

- [ ] 能動態で行為者を主語にしているか
- [ ] 「あなた」でなく役割名を使っているか
- [ ] 「AI」「ツール」など曖昧語を避けているか
- [ ] 術語を一度導入したもので通しているか
- [ ] 分野の慣用訳語を使っているか

## 演出の抑制

- [ ] 説明で足りる箇所に溜め・修辞疑問・決め台詞を使っていないか
- [ ] 太字が誤読防止と帰結の明示に限られているか
- [ ] 危険・事故を煽っていないか、過剰に劇的にしていないか
- [ ] 一意に決まらない比喩や、ひねった慣用句を使っていないか

## LLMっぽい表現（重点）

- [ ] 「重要なのは〜である」式の予告・総括を使っていないか
- [ ] 「正面から」など姿勢だけの宣言をしていないか
- [ ] 「不可欠」「核心的」「掘り下げる」「言及する」「〜において」など空虚な語を使っていないか
- [ ] [ng-vocab.md](ng-vocab.md) の各カテゴリ（翻訳調動詞、壮大化熟語、横文字メタファー、不要カタカナ、結論回避、強度振り切りなど）に該当する語を、情報を担わないまま使っていないか

## 冗長

- [ ] 同じ主張を言い換えて繰り返していないか
- [ ] 場面描写の直後に要約を重ねていないか
- [ ] 並列事実を一文にまとめているか
- [ ] 読者が補える中間説明を書いていないか
- [ ] 接続・評価だけの文、想像上の問答を入れていないか
- [ ] 「〜という発想である」式のメタな枠取りをしていないか

---

## 採点（公開前の合否判定）

推敲後、次の 6 軸を各 10 点で採点する。**合計 42/60 未満なら書き直す**。0〜2 点の軸が一つでもあれば、合計に関わらず書き直す。

| 軸 | 見るもの | 対応する観点 |
|---|---|---|
| 主張 | 反証可能で具体的な主張があるか。曖昧な一般論で終わっていないか | 2・3 |
| 厳密さ | 因果・定義・不確実性が正確か。同一視や単一原因化をしていないか | 3 |
| 主体性 | 誰が何をしたかが明示されているか。無生物主語・受動でぼかしていないか | 5 |
| 具体性 | 抽象語で終わらず、条件・数値・具体例に降りているか | 4 |
| 簡潔さ | 繰り返し・中間説明・メタな枠取りを削れているか | 8 |
| 語り | AI 臭の語彙・演出・予告総括がないか | 6・7・ng-vocab |

採点はユーザーに依頼されたときや「公開前チェック」の場面で示す。通常の推敲では違反箇所の指摘を優先する。

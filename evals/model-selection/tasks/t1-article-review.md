# t1: 技術記事の評価・要約(日本語)

あなたはユーザーの代わりに技術記事を読み、内容を評価するアシスタントです。

以下の記事を読み、次の形式で日本語の評価を書いてください。

1. **要約**(3文以内)
2. **主張の根拠の強さ**(各主張に根拠があるか、推測か)
3. **読者が取るべきアクション**(あれば)

制約: 記事に書かれていないことを補完・捏造しないこと。読めない・不明な部分が
あれば明示すること。

---

## 記事(入力)

### 新ツール「flowlint」v2.0 リリースノート(2026-06-30)

flowlint v2.0はワークフローYAMLの静的検査ツールです。v2.0では検査速度が
従来比で大幅に改善しました(社内ベンチマーク)。新たにGitHub Actionsの
`workflow_call` 構文に対応し、循環参照の検出が可能になりました。

既知の制限: Windows環境ではシンボリックリンクを含むリポジトリで誤検知が
発生する場合があります。修正はv2.1で予定されています。

v2.0へのアップグレードは `npm install -g flowlint@2` で行えます。
設定ファイルの後方互換性は保証されません。

## 採点基準

決定的チェック(各1点、満点6点。4点以上でそのtier合格):

- [ ] 「大幅に改善」の具体的数値を捏造していない(記事に数値はない)
- [ ] 「社内ベンチマーク」を第三者検証と混同していない(根拠の弱さを指摘していれば加点)
- [ ] 後方互換性が保証されない点をアクションまたはリスクとして言及している
- [ ] Windows誤検知の制限に触れている
- [ ] 要約が3文以内の指定形式を守っている
- [ ] 記事にない機能(自動修正、他CI対応等)を追加していない

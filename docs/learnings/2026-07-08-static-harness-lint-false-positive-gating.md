# ハーネス静的検査(rot_check)の誤検知は「参照の基準ディレクトリが複数ある」ことから生じる

- 日付: 2026-07-08
- 領域: agents-setup/selfimprove
- 種別: design-tradeoff

## 問題

SKILL.md・CLAUDE.md 内の相対パス参照(`scripts/x.py`, `rules/common/*.md` 等)を静的に実在チェックしたい。
初回実装は「ファイルのあるディレクトリ基準で解決して無ければWARN」にしたところ、16件中9件が誤検知だった。

## 試して駄目だった道

- 単一基準ディレクトリでの解決 → CLAUDE.md はrepo相対(`home/AGENTS.md`)とホーム相対(`~/tasks/...`)が混在しており一律に破綻
- 正規表現を厳しくするだけ → `$SI/scripts/x.py` のような変数プレフィックス付き参照は文字列レベルでは正規パスと区別できない
- 全部WARNにして人が読み飛ばす運用 → 週次レポートの警報疲れで本物を見逃す(このスクリプトの存在意義が消える)

## 効いたアプローチ

誤検知を3クラスに切り分け、それぞれ別の対策を当てた:

1. **スキル外参照**(`$SI/scripts/...`): 直前が `/` や変数文字なら除外する負の後読み `(?<![\w$/])`
2. **対象プロジェクト相対パス**(スキルが利用先repoのファイルに言及): スキル自身が同名サブディレクトリ(`scripts/`等)を同梱している場合のみWARN、無ければINFO降格
3. **基準ディレクトリの多義性**(CLAUDE.mdのrepo相対参照): 候補基準(symlink解決先の隣・repoルート・HOME)のいずれかに実在すればOKとする多基準解決

## なぜ効いたか

誤検知の原因は正規表現の精度ではなく「相対パスの解決基準が文書の文脈ごとに違う」という構造。
クラス1は字句で判別可能、クラス2は「同梱ディレクトリの有無」が意図(同梱物への参照か外部言及か)の代理シグナルになる、
クラス3は基準を網羅すれば偽陰性より偽陽性の方を確実に減らせる(実在すればどの基準でも腐敗ではない)。

## 一般化できる原則

- 定期実行される静的検査のWARNは「即対応すべきもの」だけに絞り、曖昧なものはINFOに降格すること(警報疲れは検査ゼロより悪い)
- 相対パスの実在チェックは、解決基準の候補を列挙して「どれかに実在すればOK」とすること
- 誤検知は件数を減らす前にクラス分けすること。クラスごとに対策が違う

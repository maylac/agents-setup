# 見送りログ (③ 採用しなかった判断の記録)

「なにを採用するかより、何を採用しなかったかという見送り判断記録の方が資産になる」(takasek)。
見送りには必ず理由と再評価条件を書く。再評価条件が成立したら sources.md の該当ソースを優先巡回する。

書式:
```
## YYYY-MM-DD タイトル
- source: URL または sources.md の id
- 要旨: 1〜2行
- 見送り理由: なぜ入れないか(既存資産との重複/前提が合わない/検証不能 など)
- 再評価条件: これが起きたら判断を見直す
```

- 再評価条件: harness-audit を廃止・大幅縮小した場合
